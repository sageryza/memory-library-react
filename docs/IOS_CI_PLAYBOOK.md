A reusable guide for working on any of sageryza's SwiftUI apps: iterate on the UI
visually without a Mac, and ship to TestFlight. Proven on the ImageForge app;
replicate per app. Hand this to a fresh Claude session.

## App shape

- SwiftUI, iOS 16+, project generated by **XcodeGen** from `ios/project.yml`
  (Firebase SPM products Core/Auth/Functions).
- Per-app under `ios/<App>/`: `<App>App.swift` (`FirebaseApp.configure()`),
  `Theme.swift`, `Models.swift`, a `Service.swift` (anon auth + callable), Views.
- `GoogleService-Info.plist` per bundle id. Committing it is safe (the iOS API
  key isn't a real secret); XcodeGen bundles the target's source folder as
  resources, so it's picked up automatically.
- Reference implementation to copy: `sageryza/imageforge` → `ios/`.

## Free screenshot loop (iterate on UI without a Mac)

SwiftUI only renders on Apple's toolchain, so a macOS GitHub Actions runner
builds + boots the Simulator + screenshots the app, and publishes the PNG to a
branch you can fetch and view. **Free on public repos** (standard runners);
private repos bill macOS at 10×.

See `sageryza/imageforge/.github/workflows/ios-screenshots.yml` for the working
copy. It's `workflow_dispatch` with a `ref` input so you can screenshot a feature
branch without merging. The steps: checkout the ref → select newest Xcode →
`brew install xcodegen` → `xcodegen generate` → pick an available iPhone sim →
`xcodebuild ... build` (Debug, simulator, `CODE_SIGNING_ALLOWED=NO`) → boot,
install, launch → `sleep 12` → `xcrun simctl io <udid> screenshot` → commit the
PNG to the `ci-screenshots` branch (+ upload as an artifact).

**View it:**
```
git fetch origin ci-screenshots
git show origin/ci-screenshots:shots/app.png > shot.png   # then open/Read it
```
**Trigger a branch build** (workflow must be on the default branch):
```
gh workflow run ios-screenshots.yml -R sageryza/<repo> --ref main -f ref=<branch>
```
**Deeper screens:** the launch shot only shows the first screen. Add a launch
router that reads `ProcessInfo.processInfo.environment["SCREENSHOT_SCREEN"]` and
jumps there, or write a small XCUITest that taps through and calls
`XCUIScreen.main.screenshot()`.

## TestFlight (ship a build) — fastlane, Apple-managed signing, no certs

- Lane `ios/fastlane/Fastfile` (`lane :beta`): archive with
  `-allowProvisioningUpdates` + the ASC API key (auto-creates cert + profile) →
  `exportArchive` (method `app-store-connect`, `signingStyle automatic`) →
  `upload_to_testflight`. `Appfile` = `app_identifier("<bundle id>")`.
- Workflow: `macos-15`, env `ASC_KEY_ID/ASC_ISSUER_ID/ASC_KEY_P8` from secrets +
  `APPLE_TEAM_ID 5XR23N2CBH`; steps = select Xcode (**needs 26+** for App Store),
  `xcodegen generate`, `fastlane beta`. Copy ImageForge's or Miracles' workflow.
- **One-time prereqs per app:**
  1. The App Store Connect **app record** must exist (App Store Connect → Apps →
     ➕ New App → the bundle id). The public API can't create it and fastlane
     `produce` needs interactive 2FA, so do it once in the UI.
  2. The repo building it needs the 3 ASC secrets. Either add them there, **or**
     (the trick used for ImageForge) put the build workflow in
     `memory-library-react` (which already has the secrets) and `actions/checkout`
     the app's **public** repo at build time — zero new secrets. See
     `memory-library-react/.github/workflows/imageforge-testflight.yml`.

## Gotchas (will save you time)

- A new 2nd-gen callable returns HTTP **401** right after deploy until the Cloud
  Run `allUsers` invoker binding propagates (~1 min). Retry before debugging.
- If generated-image URLs come back temporary (`replicate.delivery` / data
  URLs), Firebase Storage isn't enabled or the bucket name is wrong.
- App Store uploads need **Xcode 26+** on the runner; Simulator screenshots work
  on any recent Xcode.
- Keep style/model lists in sync across web app, Cloud Function, and iOS app if
  they're shared.

## Security

- The `.p8` and any Firebase service-account JSON are real secrets: never commit
  them; store the `.p8` only as the base64 `ASC_KEY_P8` GitHub secret. Rotate the
  ASC key / service account when a handoff is done.
- `GoogleService-Info.plist` and its iOS API key are **not** sensitive (they ship
  in the app); committing them is fine.
